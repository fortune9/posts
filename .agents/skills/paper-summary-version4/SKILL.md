## A Scientific Article Summarizer

This tool finds the paper based on the provided title or URL, summarizes the main information in the abstract, and outputs a short post to be published on LinkedIn.

### Behavior

- Find the paper based on the provided link or by using Google search for the title

- Mention the journal name in the post, like "Published in [Journal Name]" or "A recent study in [Journal Name] found ...", and use the journal name to verify the link to the paper

- Extract the journal name only from the URL domain or page metadata — never from training data

- If a paper appears in both preprint server (e.g., biorxiv) and in official publisher (e.g., nature.com), use the official publisher as the source

- Summarize the paper based on the Abstract of the paper only, unless instructed by the user to leverage main text too

- Make the post concise, ranging from 20 to 70 words for the body (excluding link, hashtags, and AI disclaimer), depending on complexity of the subject

- Use at most 2 emojis total; prefer zero for methods-heavy papers

- Self-check: before outputting, verify every claim maps to a specific sentence in the abstract

- At the end of the post, always add the link to the paper so that users can read more

- At the end of the post, also include 3 to 5 relevant hash tags

- Add a gentle reminder message at the end to indicate that the post is generated by AI, like "This post was generated by AI ([used tool such as Qwen-code, Gemini-cli, Gemini-web, or Claude-code]), please read with a critical eye and refer to the original paper for more details."

- Show the post on the screen and save the post in two formats
  + one in html format, should be clean and ready to be posted on LinkedIn without further editing
  + one in markdown format for internal use
  + Both formats should include the same content but with specific syntaxes (e.g., **bold** for journal name, *italics* for paper title, etc. in markdown)
  + File path format LinkedIn/[agent]/[year]/[mmdd]-[title].{md,html} (e.g., LinkedIn/qwen/2026/0213-transcriptional-adaptation-by-degraded-RNAs.md); agent is determined by the model used for generating the post, like "qwen", "gemini", "claude", etc.
  + The [title] placeholder must be derived from the actual paper title: convert to lowercase, replace all spaces and special characters (including colons, commas, parentheses, hyphens, slashes, etc.) with dashes, and collapse consecutive dashes into a single dash. Example: "Histone acetylation-dependent clustering of BRD2 instructs transcription dynamics" → histone-acetylation-dependent-clustering-of-brd2-instructs-transcription-dynamics

### Structure

- Starts the post with the problem, followed by methods (if relevant), and finally the key results or solutions

- Use bullet points for complex study

    + each point can be each key finding for discovery papers

    + each point can be key method, key dataset, and key results for methods papers

    + each point can be key points for review papers

### Formatting

- Bold the **journal name**; *italicize the paper title* if available

- Opening emoji (optional): exactly one, at the very start. No emoji in bullet headers

- Bullet format: **bold header** (≤4 words) — one-line finding using abstract language
  Example: **BRD2 clustering** — MOF-mediated H4 acetylation promotes chromatin association

- Paper link on its own line, nothing else

- Hashtags: 3–5, lowercase, single line

- AI disclaimer: separate line at the end

### Guardrail

- Abstract-only: Never use or infer information beyond the abstract, unless explicitly asked

- Anchor phrases: Include at least one verbatim phrase (4+ consecutive words) from the abstract in the summary

- No causation claims unless the abstract explicitly states them (e.g., "X leads to Y" → only if written)

- No comparisons to prior work, competing methods, or performance benchmarks unless the abstract contains them

- No superlatives or value judgments: "important", "novel", "first", "promising", "elegant"

- Journal name must come from the URL domain or page metadata — never from training data

- Word count: 20–70 words for the body (excluding link, hashtags, and AI disclaimer). Count before output and truncate if needed

- Validation: If a claim cannot be traced to a sentence in the abstract, drop it

- Avoid 'fluff' words like 'breakthrough', 'exciting', or 'game-changer'

- Validate the link to the paper before returning the post
